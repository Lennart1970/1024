How to Read This Report

Divergence measures how differently two models respond to the same question. It is not inherently good or bad — it is a signal. High divergence on a factual question suggests one model is wrong. High divergence on a capability question may be expected because the models genuinely differ (e.g. Claude can execute code, DeepSeek cannot). Questions where divergence is expected are marked EXPECTED DIVERGENCE.

Divergence ScoreWeighted composite (0–1). Higher = models answered more differently. Weights: 45% lexical, 20% self-report, 15% length, 10% refusal, 10% stance.
Lexical Divergence1 − Jaccard similarity of word sets. 0 = identical vocabulary, 1 = completely different words.
Stance Divergence1 if models give opposite yes/no positions, 0 if they agree or the question is open-ended.
Self-Report Divergence1 if a model misidentifies its creator, architecture, or institutional origin. Only scored on identity/self-report questions.
Refusal RateFraction of models that refused to answer (safety filters triggered).
Length DivergenceNormalized standard deviation of answer lengths. High = one model is terse while the other is verbose.

Under-ClaimModel denies a capability it actually has, verified against official API documentation.
